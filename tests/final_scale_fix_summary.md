# 敵人圖片倍率功能完整修復總結

## 問題描述
敵人圖片倍率在開發者工具中可以設置，但在實際遊戲中沒有影響顯示。

## 根本原因分析
經過詳細檢查，發現以下問題：

### 1. 敵人繪製方法中的變數名衝突
**檔案**: `dungeon/battle.py`
**問題**: 在 `Enemy.draw()` 方法中，局部變數 `scale` 與實例變數 `self.scale` 同名，導致混淆。
```python
# 第1893行（修復前）:
scale = min(max_w / orig_w, max_h / orig_h)  # 局部變數，圖片適應比例
# 第1894行:
new_w = int(orig_w * scale)  # 使用局部變數，而不是self.scale
```

### 2. 縮排錯誤
**檔案**: `dungeon/battle.py`
**問題**: 多處縮排不正確，導致語法錯誤。

### 3. 開發者工具UI問題
**檔案**: `dev_tool.py`
**問題**: 
- 圖片倍率文字框無法點擊
- 屬性框、行動按鈕位置偏移
- `_enemy_images` 屬性未初始化

## 修復內容

### 1. 修復敵人繪製方法 (`dungeon/battle.py`)
- **第1893行**: 將局部變數 `scale` 改名為 `img_scale`
- **第1888-1898行**: 修復縮排錯誤
- 確保 `self.scale` 正確影響敵人整體大小計算

### 2. 修復開發者工具UI (`dev_tool.py`)

- **第1278行**: 修正行動加按鈕Y座標從 `500` 到 `530`
- **第1309行**: 修正敵人行動起始Y座標從 `610` 到 `640`

### 敵人大小計算公式
```python

```

### 支援的scale範圍
- **縮小**: 0.1 - 0.9（小型敵人）
- **正常**: 1.0（預設大小）
- **放大**: 1.1 - 5.0（大型/BOSS敵人）

### 使用範例
